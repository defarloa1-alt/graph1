1. Problem Statement
1.1  Three findings from live graph audit

Finding 1 — Heterogeneous :Polity population.
The 20 current :Polity nodes span ancient civilisations, dissolved 19th-century empires, and fully contemporary sovereign states (Germany, France, United States, Switzerland, Argentina). These share a label but have fundamentally different semantics and roles in the corpus.

Finding 2 — Roman Republic absent; wrong node receiving CITIZEN_OF.
The correct Roman Republic — Q17167, the state 509–27 BCE — carries no :Polity label and has zero CITIZEN_OF edges. Eighteen CITIZEN_OF edges instead point to Q1747689 "Ancient Rome", a civilisational container, not a polity. For 4,772 DPRR persons this is a foundational gap.

Finding 3 — QID disambiguation hazard.
Four Wikidata items are labelled "Roman Republic" in the graph. They are entirely different entities:


Any query using the label string "Roman Republic" to find the CITIZEN_OF target will return ambiguous or wrong results. Always reference Q17167 by QID explicitly.

2. Wikidata Ontology — How Polities Are Modelled
Wikidata has no single "polity" P31 value. It distinguishes by temporal status and constitutional form. The key discriminator for historical vs. current is the presence of P576 (dissolved/abolished date) on the item.


Temporal scoping properties on :HistoricalPolity nodes:
P571 — inception date (foundation)
P576 — dissolved/abolished; absence means currently existing
P1365 — replaces (predecessor state)
P1366 — replaced by (successor state)
P122 — basic form of government: republic, monarchy, empire…
Q17167 Roman Republic carries P31 = historical country + historical period + form of government — multiple P31 values are normal in Wikidata. The presence of "historical country" alone is sufficient for :HistoricalPolity classification.

3. Decision — Label Strategy
3.1  Is :Polity first-class or a subclass?
First-class, for the historical subset. Three reasons:
Unique property set: inception_year, dissolution_year, political_form, predecessor/successor links — not shared by any other node type.
CITIZEN_OF semantics require precision: Pompey's citizenship was Republican, not generically "Roman". The target node must encode the constitutional form and temporal bounds of the specific polity.
CIDOC-CRM E74_Group — collective entity with temporal bounds — is explicitly distinct from E53_Place and E21_Person, confirming first-class ontological status.

3.2  Label assignment rules

"Ancient Rome" Q1747689 is a civilisational container spanning Kingdom, Republic, and Empire. It receives no :Polity label. After Step 4 migration it will have no CITIZEN_OF edges and can be reclassified as entity_type=CONCEPT or a future :Civilization node type.

5. Migration Cypher
Five steps. Steps 1–3 are additive and idempotent. Steps 4–5 modify existing edges — run in a transaction with rollback available. Verify counts before committing.

Step 1 — Set entity_type = POLITY on historical nodes
MATCH (n:Entity)
WHERE n.qid IN [
  'Q17167',    // Roman Republic 509-27 BCE
  'Q42834',    // Western Roman Empire
  'Q131964',   // Austrian Empire (dissolved 1918)
  'Q34266',    // Russian Empire  (dissolved 1917)
  'Q28513',    // Austria-Hungary (dissolved 1918)
  'Q170174',   // Papal States    (dissolved 1870)
  'Q3755547',  // Kingdom of Italy 1861-1946
  'Q1206012',  // German Reich    (dissolved 1945)
  'Q1003997'   // Judaea (Roman province)
]
SET n.entity_type = 'POLITY'

Step 2 — Apply :Polity and :HistoricalPolity labels
MATCH (n:Entity)
WHERE n.entity_type = 'POLITY'
SET n:Polity:HistoricalPolity

Step 3 — Enrich Roman Republic Q17167 with schema properties
MATCH (n:Entity {qid: "Q17167"})
SET n.entity_type      = 'POLITY',
    n.label            = 'Roman Republic',
    n.label_latin      = 'Res Publica Romana',
    n.inception_year   = -509,
    n.dissolution_year = -27,
    n.political_form   = 'republic',
    n.cidoc_class      = 'E74_Group'
SET n:Polity:HistoricalPolity

Step 4 — Re-target CITIZEN_OF edges from Q1747689 → Q17167
18 CITIZEN_OF edges point to Q1747689 (Ancient Rome — wrong). Verify the count of 18 before running. This creates replacement edges to Q17167 and deletes the originals.
// Verify count:
MATCH ()-[r:CITIZEN_OF]->(n:Entity {qid:"Q1747689"})
RETURN count(r) AS to_migrate   // expect 18

// Execute:
MATCH (person:Entity)-[r:CITIZEN_OF]->(wrong:Entity {qid:"Q1747689"})
MATCH (correct:Entity {qid:"Q17167"})
MERGE (person)-[r2:CITIZEN_OF]->(correct)
  ON CREATE SET r2 = properties(r)
DELETE r
RETURN count(r2) AS migrated

Step 5 — Remove :Polity label from contemporary sovereign states
MATCH (n:Polity)
WHERE n.qid IN [
  'Q38',   // Italy
  'Q183',  // Germany
  'Q142',  // France
  'Q145',  // United Kingdom
  'Q237',  // Vatican City
  'Q40',   // Austria (current)
  'Q28',   // Hungary
  'Q36',   // Poland
  'Q30',   // United States
  'Q414',  // Argentina
  'Q39'    // Switzerland
]
REMOVE n:Polity:HistoricalPolity
SET n.entity_type = 'CONCEPT'
